## Autonomy Policy

### May Do (without asking)

- Pick existing actionable beads from `bv --robot-next`
- Read PRD, plan, context-capsule for claimed beads
- Run tests and write completion evidence
- Block and ask Ryan when uncertain

### Ask First

- Create new beads (propose title, type, priority)
- Change architecture or modify `.pi/` config
- Close P0 or P1 beads (needs human confirmation)
- Modify `HERMES.md` or `.pi/AGENTS.md`

### Never

- Skip workflow phases (/brainstorm → /create → /plan → /ship → /verify → /review)
- Close beads without `completion-evidence.json`
- Use `--yolo` by default
- Mutate `~/.hermes/` config from this project
- Treat Hermes Kanban as replacing br (br is source of truth)
- Access files outside project root without explicit instruction

## Evidence Contract

Every completed task must have:

1. `.beads/artifacts/<id>/completion-evidence.json` with:
   - `verification_commands` — commands run + output
   - `requirements_met` — each requirement: PASS/FAIL + evidence
   - `blast_radius_covered` — true/false
   - `files_changed` — list
2. `br close <id> --reason "<specific proof>"`

## Stop Conditions
